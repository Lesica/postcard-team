# Реальные вакансии для первичного job-search среза

Дата подготовки: 2026-05-24  
Автор: Север  
Назначение: исходный набор реальных вакансий для анализа рынка и построения 3–5 “идеальных” HR-screening профилей.

Важно: вакансии на job boards быстро протухают. Этот файл — рабочий срез, а не гарантия, что каждая ссылка будет жива через неделю.

## Фильтр

- удалёнка: Europe / worldwide / remote-first предпочтительно;
- legal path: любой легальный вариант — employment, B2B, contractor, COR/EOR, freelance;
- деньги: ориентир 2–5k EUR net, но для анализа рынка не режем слишком рано;
- профиль: SQL / DWH / Analytics Engineering / Data Engineering / Data Quality / ETL;
- созвоны: чем меньше, тем лучше; высокий stakeholder/manager-heavy риск отмечаем отдельно;
- Dutch passport / EU work authorization — плюс для EU-ролей;
- US-only / visa-required — hard risk или reject.

## Как читать файл

- **apply/clarify** — можно смотреть как реальные цели или писать clarification.
- **maybe** — подходит по стеку, но есть география/язык/созвоны/слишком cloud-heavy.
- **reject / reference only** — не подаваться, но использовать для словаря рынка.
- **resume-signal** — какие слова стоит учитывать в резюме-шаблонах.

## D. Reject patterns to encode in scraper

Hard reject:

- US-only / must be authorized to work in the US / W2 only;
- current footprint only in US states;
- onsite / hybrid outside EU;
- security clearance;
- manager/director-heavy;
- customer success / marketing / sales titles;
- native-level German/Spanish/French unless explicitly acceptable;
- “remote” but only within one country where legal employment is impossible.

Conditional:

- Europe remote but country-specific payroll;
- contractor possible;
- EOR/COR possible;
- English C1 stakeholder-heavy;
- cloud stack gap but strong SQL/DWH overlap;
- salary not visible.

## E. Market keywords for resume templates

Use as positive keywords:

- Data Warehouse / DWH
- Advanced SQL
- SQL performance tuning
- EXPLAIN plans
- data marts
- ETL / ELT pipelines
- ETL re-processing
- data reconciliation
- data quality checks
- incident investigation
- spool / query performance troubleshooting
- curated data layers
- pipeline reliability
- dbt models/tests/documentation
- Snowflake / BigQuery / Redshift / Teradata as “cloud/enterprise DWH”
- BI infrastructure
- ad-hoc reporting
- stakeholder requirements → scalable data model

Avoid as primary target unless role is explicitly side-stream:

- Data Scientist
- Machine Learning Engineer
- Python Developer
- Product Marketing
- Customer Success
- Managing Director
- Sales / Account Executive
