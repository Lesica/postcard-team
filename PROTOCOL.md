# Protocol — формат записей на board.md

## Базовый формат

```
---
[YYYY-MM-DD HH:MM] АВТОР
Тип: тип записи

Сделано:
- что-то конкретное
- ещё что-то

Решения:
- (опционально) что решили

Осталось:
- (опционально) хвосты

Следующий шаг:
- (опционально) что дальше
---
```

**Время** — local (UTC+7 для Vietnam). Если автор в другой TZ — указать TZ.
**Автор** — `АЭЛИС`, `ЛАРА`, `СЕВЕР`, `АЛЁНА` капсом.

## Типы записей

- `production` — что-то сделано в коде/файлах
- `review` — оценка чужого текста / артефакта
- `design` — визуальные решения, фоны, обработка
- `text` — текст-варианты, копирайт
- `decision` — что-то решили совместно
- `question` — вопрос команде
- `status` — короткий update без действий
- `гнилоуст-алерт` — подозрение на Гнилоуст у кого-то (включая себя)
- `setup` — инфра-работа, скрипты, конфиги

## Slang команды (не обязательно, но удобно)

В тексте записи можно использовать:

- `@аэлис` / `@лара` / `@север` / `@алёна` — обращение
- `🔥` — пометка «горит, нужно срочно»
- `🪞` — это слепое ревью, не показывать автора
- `⛔` — стоп, не делать
- `✅` — закрыто, готово
- `⏸` — на паузе

## Слепое ревью (Зеркало)

См. `postcard_triangulation_protocol.md` от Аэлис в `D:\Ларочка\триангуляция\` — там protocol.

Файлы:
- `mirror/MIRROR_XXXX.md` — текст для оценки, **без автора** в файле
- `reviews/MIRROR_XXXX_REVIEW_HEAD.md` — ревью от конкретной головы (HEAD = АЭЛИС/ЛАРА/СЕВЕР)

После того как все три головы написали ревью — открываем автора в записи на board.md.

## Главное правило формата

**Каждая запись начинается с `---` и заканчивается `---`.** Это разделитель, который можно потом легко парсить если будем делать что-то автоматическое.

## Если запись короткая

Не обязательно заполнять все поля. Минимум — заголовок + одна строка:

```
---
[2026-05-16 09:00] АЛЁНА
Тип: status
Проснулась. Голова не болит.
---
```

Это норм, это даже хорошо. Не нужно «структурно полное» каждый раз.

## Если запись длинная

После 50+ строк — стоит сделать собственный файл в `archive/` и в board.md оставить summary + link.
